## 프로젝트 개요

### 프로젝트명
**해외송금 수수료 비교** (Cross-border Remittance Lookup)

### 목적
한국에서 해외로 송금할 때 8개 주요 서비스(Wise, SentBe, MOIN, WireBarley, 토스, PayPal, 하나은행, 신한은행)의 실제 수수료와 환율을 실시간으로 비교하여, 사용자가 가장 저렴한 방법으로 송금할 수 있도록 돕는 웹/앱 서비스입니다.

### 타겟 사용자
- **유학생 및 학부모**: 학비, 생활비 송금
- **해외 거주 한국인**: 가족 생활비, 부모님 용돈 송금
- **외국인 노동자**: 본국 가족에게 급여 송금
- **기업**: 수입/수출 대금 결제
- **프리랜서**: 해외 클라이언트로부터의 급여 수령

### 문제 인식
1. **정보 비대칭**: 각 서비스의 수수료와 환율을 일일이 확인하기 어려움
2. **숨겨진 비용**: 표면적 수수료 외에 환율 스프레드(마진)가 숨어있음
3. **시간 소모**: 여러 앱/웹사이트를 방문하여 비교해야 함
4. **불투명성**: 실제 받는 금액을 미리 알기 어려움

### 솔루션
- **원스톱 비교**: 8개 서비스를 한 화면에서 비교
- **투명한 비용**: 수수료 + 환율 스프레드 + 실수령액을 모두 표시
- **실시간 데이터**: 주 2회 자동 업데이트 (화/금 오전 9시)
- **무료 서비스**: 사용자 가입/로그인 불필요, 완전 무료

## 핵심 가치 제안

### 1. 투명성 (Transparency)
- 수수료와 환율 스프레드를 명확하게 분리 표시
- 실제 받는 금액을 원화 기준으로 계산
- 숨겨진 비용 없이 모든 정보 공개

### 2. 편의성 (Convenience)
- 가입/로그인 불필요
- 모바일/데스크톱 최적화
- PWA로 앱처럼 사용 가능
- 퀵 버튼으로 빠른 금액 입력 (100만원, 500만원 등)

### 3. 정확성 (Accuracy)
- 실제 API 데이터 사용 (Wise, PayPal)
- 수동 검증된 한국 서비스 데이터
- 주 2회 자동 업데이트로 최신 정보 유지

### 4. 완전 무료 (Free)
- **사용자**: 완전 무료 (가입 불필요)
- **운영비**: ₩0/월 (GitHub Actions + Firebase 무료 티어)

## 주요 기능

### 1. 실시간 환율 비교
**8개 통화 지원**
- 🇺🇸 USD (미국 달러)
- 🇯🇵 JPY (일본 엔, 100엔 기준)
- 🇪🇺 EUR (유로)
- 🇬🇧 GBP (영국 파운드)
- 🇨🇳 CNY (중국 위안)
- 🇦🇺 AUD (호주 달러)
- 🇨🇦 CAD (캐나다 달러)
- 🇸🇬 SGD (싱가포르 달러)

**8개 송금 서비스**
- **핀테크**: Wise, SentBe, MOIN, WireBarley, 토스, PayPal
- **은행**: 하나은행, 신한은행

### 2. 목적별 최적 서비스 추천
**송금 목적에 따른 필터링**
- 💼 **유학비 (학비)**: 고액 송금, 수수료 절감 중요
- 🏠 **생활비**: 중액 송금, 빠른 처리 중요
- 💰 **외국인 노동자 급여**: 소액 송금, 간편함 중요
- 🏢 **수입/수출 대금**: 대액 송금, 안정성 중요
- 🎁 **기타**: 일반 송금

### 3. 상세 비용 분석
각 서비스별로 표시:
- **중간환율**: 공정한 기준 환율 (open.er-api.com)
- **적용환율**: 실제 적용되는 환율
- **환율 스프레드**: 중간환율과의 차이 (%)
- **송금 수수료**: 고정 수수료 (₩)
- **총 비용**: 수수료 + 환율 마진
- **실수령액**: 외화로 받는 금액
- **절약 금액**: 최저가 대비 얼마나 비싼지

### 4. 히스토리 차트 (2020-2026)
- **6년 환율 데이터**: 월별 환율 변동 추이
- **8개 통화 비교**: 멀티 차트로 한눈에 비교
- **고점/저점 분석**: 현재 환율이 역사적으로 높은지/낮은지
- **송금 타이밍**: 유리한 송금 시점 판단 가능

### 5. 서비스 가용성 표시
- **24시간 신청 가능**: Wise, SentBe, MOIN, WireBarley, 토스, PayPal
- **영업일만 가능**: 하나은행, 신한은행 (09:00~16:00)
- **주말/공휴일**: 핀테크는 신청 가능, 은행은 영업일 처리

### 6. 프로모션 정보
각 서비스의 현재 프로모션:
- SentBe: 가입 시 무료쿠폰
- MOIN: 환율우대 100%
- WireBarley: 50만원 이상 수수료 무료
- 하나은행: 인터넷뱅킹 환율우대 50%
- 신한은행: 인터넷뱅킹 수수료 50% 할인

### 7. 반응형 디자인
- **모바일 최적화**: 터치 인터페이스, 스와이프 동작
- **태블릿**: 최적화된 레이아웃
- **데스크톱**: 넓은 화면에서 더 많은 정보 표시
- **다크 모드**: 눈의 피로 감소

## 데이터 자동화

### 자동 업데이트 시스템

**일정**
- **요일**: 매주 화요일, 금요일
- **시간**: 오전 9:00 (KST) = 00:00 (UTC)
- **Cron**: `0 0 * * 2,5`

**프로세스**
1. **환율 수집** (5초)
   - open.er-api.com/v6/latest/KRW
   - 8개 통화 동시 수집
   - 중간환율 기준

2. **수수료 수집** (10초)
   - Wise Comparison API (Wise, PayPal)
   - fixed-fees.json (한국 서비스 6개)
   - 환율 스프레드, 송금 시간, 프로모션

3. **데이터 검증** (5초)
   - 필수 필드 존재 여부
   - 숫자 유효성 검증
   - 이상치 감지 (전주 대비 10% 이상 변동)

4. **파일 생성** (2초)
   - fee-data.json 업데이트
   - 변경사항 있을 때만 진행
   - 타임스탬프 추가

5. **커밋 & 푸시** (5초)
   - Git commit -m "chore: update remittance data"
   - Git push origin main

6. **빌드** (30초)
   - npm run build
   - Vite 최적화 빌드
   - 번들 크기: ~614KB

7. **배포** (10초)
   - firebase deploy --only hosting
   - CDN 캐시 무효화
   - 전 세계 동시 배포

**총 소요 시간**: ~67초 (1분 10초)

### 에러 처리

**자동 복구**
- API 일시 실패 → 기존 데이터 유지, 다음 스케줄 재시도
- 일부 통화 실패 → 성공한 통화만 업데이트
- 변경사항 없음 → 배포 건너뛰기

**알림 시스템**
- GitHub Actions 실패 → 자동 Issue 생성
- 제목: "⚠️ Automated Data Update Failed"
- 레이블: `automation`, `bug`
- 내용: 실패 시간, 로그 링크, 해결 방법

### 수동 업데이트

**GitHub UI**
1. Actions 탭 → "Update Remittance Data & Deploy"
2. "Run workflow" 버튼 클릭

**로컬 실행**
```bash
node scripts/update-fees.mjs
git add public/fee-data.json
git commit -m "chore: manual data update"
git push
npm run deploy
```

## PWA 구현

### Progressive Web App 특징

**1. 설치 가능 (Installable)**
- Android/iOS 홈 화면에 추가
- 브라우저 UI 없이 전체 화면 실행
- 네이티브 앱과 동일한 사용자 경험

**2. 오프라인 지원 (Offline)**
- Service Worker로 정적 파일 캐싱
- 네트워크 없어도 기본 기능 사용 가능
- 마지막 데이터로 비교 가능

**3. 자동 업데이트 (Auto-update)**
- 화/금 업데이트 즉시 감지
- "새 버전 사용 가능" 알림 표시
- 원클릭으로 최신 버전 적용

**4. 빠른 로딩 (Fast)**
- Cache First: 정적 파일 즉시 로딩 (~0.5초)
- Network First: 데이터는 최신 유지
- 재방문 시 2-3배 빠른 속도

### 기술 구현

**Service Worker (`/public/sw.js`)**
```javascript
// 캐싱 전략
- Cache First: HTML, CSS, JS → 빠른 로딩
- Network First: fee-data.json → 최신 데이터
- 자동 업데이트: 새 버전 감지 시 알림
```

**Web App Manifest (`/public/manifest.json`)**
```json
{
  "name": "해외송금 수수료 비교",
  "short_name": "해외송금비교",
  "display": "standalone",
  "icons": [192x192, 512x512],
  "background_color": "#09090B",
  "theme_color": "#09090B"
}
```

**PWA 등록 (`/public/pwa-register.js`)**
- Service Worker 자동 등록
- 설치 안내 배너 표시 (Android)
- 업데이트 알림 UI
- Google Analytics 연동

### PWA vs 네이티브 앱

## 기술적 도전과 해결

### 도전 1: 실시간 데이터 수집
**문제**: 각 서비스의 API가 없거나 인증 필요
**해결**:
- Wise: 공개 Comparison API 활용
- 한국 서비스: 수동 검증 후 fixed-fees.json 관리
- 자동화: GitHub Actions로 주 2회 업데이트

### 도전 2: 비용 절감
**문제**: 서버 운영비, API 비용 부담
**해결**:
- 무료 API만 사용 (open.er-api.com, Wise API)
- Firebase 무료 티어 활용
- GitHub Actions 무료 티어 활용
- **결과**: 월 ₩0 운영비

### 도전 3: 모바일 UX
**문제**: 좁은 화면에서 많은 정보 표시
**해결**:
- 반응형 디자인 (clamp 함수)
- 스와이프로 통화 전환
- 접기/펼치기 UI
- PWA로 전체 화면 활용

### 도전 4: 데이터 신뢰성
**문제**: API 실패, 이상치 데이터
**해결**:
- 데이터 검증 로직
- 이상치 감지 (전주 대비 10% 초과)
- 실패 시 기존 데이터 유지
- 자동 알림 (GitHub Issue)
